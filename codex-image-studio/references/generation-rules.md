# Generation Rules

## Goal

Define how `codex-image-studio` should turn learned behavior models into image-first post outputs.

## Local Render Target

Default image API target:

- `http://107.172.148.170:8000/v1/images/generations`

Default model:

- `gpt-image-2`

Auth:

- `OPENAI_API_KEY`

## Core Generation Principle

Generate from behavior systems, not from one-to-one copying.

The downstream content should inherit:

- promise grammar
- educational structure
- visual archetype
- funnel logic

without mechanically cloning a single reference post.

## Priority Order

1. preserve account behavior DNA
2. preserve title/copy/visual consistency
3. adapt topic to the user's intent
4. keep outputs machine-readable

## Title Rules

Generate 3-10 title candidates depending on request.

Preserve:

- time compression if the source account uses it
- practical promise style
- low-barrier framing
- directness or mystical tone depending on the account

## Copy Rules

Generate copy with:

- strong opening
- clear structure
- controlled density
- compatible CTA

If the source account uses instructional breakdowns, keep that structure.

## Image Prompt Rules

Each prompt should explicitly encode:

- style archetype
- palette
- material/background texture
- border/decorative grammar
- typography direction
- core subject
- density level
- composition layout

Do not leave the prompt at generic aesthetic adjectives only.

## Safety / Originality Constraint

Do not reproduce:

- exact slogans when they are highly distinctive
- exact branded signatures
- literal account names unless the user explicitly wants a benchmark mock
- copyrighted logos or watermarks

Transfer the system, not the fingerprint.
